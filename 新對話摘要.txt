## 技術架構

### 前端（GitHub Pages）
純靜態 HTML + JS，無框架。

**重要限制：**
- Safari 不支援 AbortSignal.timeout()：所有 fetch 不可加此參數
- supabase-js 在 GitHub Pages 環境 apikey header 帶不進去：新頁面用直接 fetch；舊頁面需在 head 加 CDN：
  ```html
  <script src="https://cdn.jsdelivr.net/npm/@supabase/supabase-js@2.39.3/dist/umd/supabase.js"></script>
  <script src="js/supabase.js"></script>
  ```
- 新頁面（radar.html、news.html、daily.html 強力買入區）使用直接 fetch

**Supabase 直接 fetch 標準寫法：**
```javascript
const SUPABASE_URL = 'https://kldumnxdlgovaxtiszyn.supabase.co';
const SUPABASE_ANON = 'eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...';
const SB_HEADERS = {
  'apikey': SUPABASE_ANON,
  'Authorization': 'Bearer ' + SUPABASE_ANON,
};
```

### 後端（Railway Flask）

| 路由 | 功能 |
|------|------|
| /webhook | TradingView 快訊接收（指標通知已停用） |
| /quote | 股票報價 API（台股自動判斷 .TW / .TWO） |
| /sec13f | SEC EDGAR 機構持股查詢（yfinance institutional_holders + EDGAR 總機構數） |
| /test | 健康檢查 |
| /state | 查看狀態 |

**/quote 回傳欄位：** name, price, currency, closes, highs, lows, previousClose, sector, industry, forwardEps, trailingEps, trailingPE, forwardPE, pegRatio, earningsGrowth, industryPE, targetMeanPrice, targetLowPrice, targetHighPrice, recommendationKey, numberOfAnalystOpinions

**/sec13f 回傳欄位：** symbol, total_holders（EDGAR統計）, top10（前10大機構名單）, pct_institutional

## 美股強力買入掃描系統（daily.html）

### 頁籤
1. 📊 盤面解析 — 原有功能
2. 🎯 強力買入清單 — 每日自動掃描結果

### 篩選 chips
- 全部
- 強力買入（grade = 強力買入）
- 買入（grade = 買入）
- 💎 低估（PEG低估 + PE偏低 + 上漲空間 25-30%）
- 💎💎 嚴重低估（PEG低估 + PE偏低 + 上漲空間 ≥ 30%）

### 評分邏輯（daily_scanner.py）
| 維度 | 滿分 | 說明 |
|------|------|------|
| 技術面 | 25 | MA黃金交叉、RSI健康區間、接近支撐 |
| 分析師 | 25 | 券商評級、分析師數量、目標價上漲空間 |
| 法人籌碼 | 25 | SEC EDGAR 機構數、前10大持倉金額、大機構持有 |
| 基本面 | 25 | PEG、預估PE、EPS成長 |

### 評級門檻
- 強力買入：≥ 80分
- 買入：≥ 65分
- 觀察：≥ 50分
- 暫不推薦：< 50分

### Admin 管理
- 在 daily.html 的「強力買入清單」頁籤，admin 登入後可看到管理區
- 可新增/刪除 scan_watchlist 的股票代號

## 台股主力雷達（radar.html）

### 三個頁籤
1. 主力新建倉 — 本月相較上月新增個股 + 外資連續買超
2. 主力持股追蹤 — 多方共識 / 依基金查詢 / 法人清單
3. 主力ETF追蹤 — 投信持有 ETF 清單

### taiwan_scanner.py 重要邏輯
- ETF 過濾：is_etf() 函式（/^0\d{3,5}[A-Za-z]?$/）
- 歷史補抓：--month YYYYMM，強制重爬（if check.data and not target_month）
- 比對月份：get_prev_months(current_month, n=1)
