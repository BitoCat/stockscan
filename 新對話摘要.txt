## TradingView 快訊中繼 Bot（tw-alert-bot）✅ 2026-06-07 完成

### 目的
TradingView 快訊只送股票代號（如 `2330`），中繼 Bot 自動補上台股中文名稱後送到 Telegram。美股維持原代號不轉換。

### 架構
```
TradingView Webhook → Railway（FastAPI）→ Supabase tw_stocks → Telegram
```

### 關鍵資訊
| 項目 | 內容 |
|------|------|
| GitHub | https://github.com/BitoCat/tw-alert-bot |
| Railway Webhook URL | https://tw-alert-bot-production.up.railway.app/webhook |
| Railway Health Check | https://tw-alert-bot-production.up.railway.app/health |
| 本機專案位置 | `~/Downloads/home/claude/tw-alert-bot/` |

### Railway 環境變數
- `SUPABASE_URL`：同交易大師專案
- `SUPABASE_KEY`：**service_role key**（不是 anon key，因為需要寫入權限）
- `TELEGRAM_TOKEN`：Telegram Bot Token

### Supabase 資料表
- 資料表：`tw_stocks`（欄位：`code` PK、`name`、`updated_at`）
- 資料來源：TWSE Open API（上市）+ TPEX Open API（上櫃）
- 共 2249 筆

### TradingView 快訊 JSON 格式
```json
{
  "chat_id": "頻道ID",
  "text": "🚨: {{ticker}} 15m 低點下軌"
}
```

### 使用方式
- **早上**：開兩個台股指標快訊，對應台股清單，Webhook 打到上方 URL
- **晚上**：關台股，開兩個美股指標快訊，對應美股清單，Webhook 同一個 URL
- 切換時只在 TradingView 操作，server 完全不用動

### 運作邏輯
- ticker 是純數字（如 `2330`）→ 查 `tw_stocks`，補中文名稱
- ticker 有 `TWSE:` 或 `TPEX:` 前綴 → 去掉前綴後查表
- ticker 是美股（NASDAQ/NYSE/AMEX）→ 直接送出代號，不查表

### Telegram 頻道
- 台股報報：台股快訊頻道
- 美股報報：美股快訊頻道

### 日常維護
```bash
# 每季更新台股名稱（新股上市時）
cd ~/Downloads/home/claude/tw-alert-bot
python3 import_stocks.py

# 修改程式後部署
git add . && git commit -m "說明" && git push
# push 後 Railway 自動重新部署
```

## 技術架構

### 前端（GitHub Pages）
純靜態 HTML + JS，無框架。

**重要限制：**
- Safari 不支援 AbortSignal.timeout()：所有 fetch 不可加此參數
- supabase-js 在 GitHub Pages 環境 apikey header 帶不進去：新頁面用直接 fetch；舊頁面需在 head 加 CDN：
  ```html
  <script src="https://cdn.jsdelivr.net/npm/@supabase/supabase-js@2.39.3/dist/umd/supabase.js"></script>
  <script src="js/supabase.js"></script>
  ```
- 新頁面（radar.html、news.html、daily.html 強力買入區）使用直接 fetch

**Supabase 直接 fetch 標準寫法：**

## 概念股雷達（concepts.html）✅ 2026-06-11 完成（台股+美股）

### 架構
concepts.html（🇹🇼/🇺🇸 切換頁籤）
→ data/concepts.json（台股 20 概念約 150 檔）
→ data/us_concepts.json（美股 18 概念約 100 檔）
→ Railway POST /concept-analysis（批次估值，concept_analysis.py Blueprint）

### 概念清單維護（不用動程式）
- 直接編輯 JSON：台股 data/concepts.json、美股 data/us_concepts.json
- 台股 code 只放四碼數字（後端自動試 .TW 再 .TWO）；美股直接放代號（NVDA）
- category 可自由新增，前端自動產生新分組按鈕
- 金融類概念加 "valuation_hint": "pb"（台股金融股、美國銀行股已標）
- JSON 改壞時整片按鈕會消失 → 用 jsonlint.com 驗證（常見錯誤：多/少逗號、全形引號）
- 美股特殊代號：Block=XYZ（原SQ）、SanDisk=SNDK（2025從威騰分拆）

### 後端（trading-bot/concept_analysis.py，Flask Blueprint）
- 註冊：主程式 app = Flask 下方 from concept_analysis import concept_bp + register_blueprint
- POST /concept-analysis：body {stocks:[{code,name}], valuation_hint}，一次最多15檔，
  5線程並行，記憶體快取10分鐘
- GET /concept-analysis/test?code=XXXX：瀏覽器單檔測試（回傳含 "ver" 版本標記）
- _resolve：code 含英文字母 → 美股直接用原代號；純數字 → 台股試 .TW/.TWO
- 四種估值法：analyst（分析師目標價）、pe（近4年歷史PE區間×目前EPS）、
  pb（近4年歷史PB區間×目前每股淨值）、dividend（近5年均股利×16/20/32）
- 股利法過濾：dividend fair < 現價25% 時略過（NVDA 這類低殖利率成長股不適用）
- fair_basis 優先序：hint=pb → 分析師≥3位 → pe/pb 中位數(history) → dividend
- divergence_pct：各方法合理價分歧度，>80% 前端顯示 ⚠️（估值重評中的成長股）

### 前端（concepts.html）
- supabase CDN 做 auth + plan 檢查（free 鎖定），分析資料直接 fetch 後端
- 🇹🇼/🇺🇸 頁籤切換載入對應 JSON；概念按鈕牆依 category 分組
- 表格預設按 upside_pct 排序，欄位可點擊排序；合理價旁標依據（法人/淨值/歷史/股利）+ ⚠️

- 上方阻力/下方支撐 = 近60交易日高低點；含全站共用底部 tabbar（概念股設 active）
- 首頁 index.html quick-btns 已加 🧩 概念股雷達入口
